# 文件扩展名修复说明

## 问题描述

Git 仓库中的 `lifeimages25/` 目录下，有些文件的扩展名为大写（如 `.JPG`, `.MP4`, `.MOV`），而本地文件系统上的文件扩展名都是小写。这导致本地和远程仓库不同步。

## 解决方案

使用 `fix_git_extensions.py` 脚本将所有文件扩展名统一为小写。

## 使用方法

```bash
python fix_git_extensions.py
```

脚本会：
1. 检查 Git 仓库中所有文件的扩展名
2. 找出所有需要重命名的文件（扩展名为大写）
3. 使用两步重命名方法（Windows 文件系统不区分大小写）
4. 将所有扩展名改为小写

## 提交更改

运行脚本后，使用以下命令提交更改：

```bash
# 查看更改
git status

# 提交更改
git commit -m "统一文件扩展名为小写"

# 推送到远程仓库
git push origin main
```

## 注意事项

- 在 Windows 上，由于文件系统不区分大小写，需要使用两步重命名（先重命名为临时名称，再重命名为目标名称）
- 脚本会自动处理所有需要重命名的文件
- 重命名后，HTML 文件中的引用也会自动匹配（因为文件系统不区分大小写）
